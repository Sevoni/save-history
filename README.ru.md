# Save History

[English version](README.md) | [Español](README.es.md)

Плагин для Obsidian для сохранения и восстановления версий файлов.

## Возможности

- **Ручное сохранение версий** — сохраняйте текущее состояние файла в любой момент
- **Восстановление версий** — восстанавливайте любую ранее сохранённую версию одним кликом
- **Бэкап перед восстановлением** — перед восстановлением текущее состояние автоматически сохраняется как резервная копия
- **Сравнение изменений** — сравнивайте любые две версии или версию с текущим файлом (с подсветкой на уровне символов)
- **Просмотр версий** — просмотр версии перед восстановлением (перетаскиваемое и изменяемое по размеру окно)
- **Группировка версий** — группировка по дням, неделям, месяцам или годам (сворачиваемые группы)
- **Переименование меток** — давайте сохранённым версиям собственные имена
- **Массовое удаление** — выбирайте и удаляйте несколько версий сразу
- **Экспорт / Импорт** — экспортируйте одну или все версии файла; импортируйте ранее экспортированные версии
- **Автосохранение** — автоматическое сохранение версий с настраиваемым интервалом
- **Автосохранение при закрытии вкладки** — сохраняйте версию при закрытии вкладки файла
- **Настройки для каждого файла** — переопределяйте интервал автосохранения, автосохранение при закрытии, максимум автосохранённых версий и группировку для каждого файла
- **Максимум автосохранённых версий** — ограничьте количество хранимых автосохранённых версий на файл
- **Разрешённые расширения файлов** — выбирайте, какие типы файлов отслеживать (`.md` всегда включён)
- **Мультиязычность** — интерфейс на английском, русском и испанском языках
- **Настраиваемая папка снимков** — изменяйте место хранения версий в хранилище
- **Настраиваемая папка экспорта** — резервная папка для экспорта, если выбор папки браузером недоступен
- **Отслеживание переименований** — снимки следуют за переименованными файлами и папками
- **Очистка при удалении файлов** — снимки автоматически удаляются при удалении файла
- **Иконка на ленте** — быстрый доступ к боковой панели истории
- **Команда восстановления последней несохранённой версии** — быстрое восстановление бэкапа перед восстановлением

## Установка

### Из GitHub Releases

1. Скачайте последний zip-архив из [Releases](https://github.com/Sevoni/save-history/releases)
2. Распакуйте `main.js` и `manifest.json` в папку `.obsidian/plugins/save history/` вашего хранилища
3. Включите плагин в настройках Obsidian → Сообщественные плагины

### Вручную

1. Клонируйте этот репозиторий
2. Выполните `npm install`
3. Выполните `npm run build`
4. Скопируйте `main.js` и `manifest.json` в папку `.obsidian/plugins/save history/` вашего хранилища

## Использование

- Откройте markdown-файл в Obsidian
- Используйте боковую панель «История файла» для просмотра и управления версиями
- Или используйте команды из палитры команд:
  - **Сохранить версию** — сохранить текущее состояние
  - **Восстановить версию** — открыть окно восстановления
  - **Открыть боковую панель истории** — открыть панель со списком версий
  - **Восстановить последнюю несохранённую версию** — восстановить бэкап перед восстановлением

## Разработка

```bash
npm install
npm run build
```

Для отслеживания изменений:

```bash
npm run watch
```
