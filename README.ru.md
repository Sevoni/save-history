# Save History

Плагин для Obsidian для сохранения и восстановления версий markdown-файлов.

## Возможности

- **Ручное сохранение версий** — сохраняйте текущее состояние файла в любой момент
- **Восстановление версий** — восстанавливайте любую ранее сохранённую версию одним кликом
- **Бэкап перед восстановлением** — перед восстановлением текущее состояние автоматически сохраняется как резервная копия
- **Сравнение изменений** — сравнивайте любые две версии или версию с текущим файлом
- **Просмотр версий** — просмотр версии перед восстановлением (перетаскиваемое и изменяемое по размеру окно)
- **Группировка версий** — группировка по дням, неделям, месяцам или годам
- **Переименование меток** — давайте сохранённым версиям собственные имена
- **Мультиязычность** — интерфейс на английском и русском языках
- **Настраиваемая папка снимков** — изменяйте место хранения версий
- **Удаление с подтверждением** — безопасное удаление с запросом подтверждения

## Установка

### Из GitHub Releases

1. Скачайте последний zip-архив из [Releases](https://github.com/Sevoni/save-history/releases)
2. Распакуйте `main.js` и `manifest.json` в папку `.obsidian/plugins/save history/` вашего хранилища
3. Включите плагин в настройках Obsidian → Сообщественные плагины

### Вручную

1. Клонируйте этот репозиторий
2. Выполните `npm install`
3. Выполните `npm run build`
4. Скопируйте `main.js` и `manifest.json` в папку `.obsidian/plugins/save history/` вашего хранилища

## Использование

- Откройте markdown-файл в Obsidian
- Используйте боковую панель «История файла» для просмотра и управления версиями
- Или используйте команды из палитры команд:
  - **Сохранить версию** — сохранить текущее состояние
  - **Восстановить версию** — открыть окно восстановления
  - **Открыть боковую панель истории** — открыть панель со списком версий
  - **Восстановить последнюю несохранённую версию** — восстановить бэкап перед восстановлением

## Разработка

```bash
npm install
npm run build
```

Для отслеживания изменений:

```bash
npm run watch
```
